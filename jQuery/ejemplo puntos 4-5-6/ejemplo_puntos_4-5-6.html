Indica los eventos estándares en el DOM

Los eventos estándares del DOM pueden clasificarse en cuatro grupos: eventos de ratón, de teclado, html, y de DOM (o mde mutación). También existen eventos específicos de distintos navegadores (IE o Mozilla) y para navegadores que trabajan en dispositivos con pantallas táctiles.

Ejemplos de los más frecuentes:

Eventos de Teclado

  • keydown: Se dispara cuando se pulsa una tecla
  • keypress: Se dispara cuando se presiona una tecla
  • keyup: Se dispara cuando se deja de presionar una tecla

Eventos de ratón

  • click: Se dispara al hacer click en un elemento
  • dblclick: Se dispara al hacer doble click en un elemento
  • mousedown: Se dispara al pulsarse en un elemento
  • mouseup: Se dispara al levantarse el botón tras hacer mousedown
  • mouseover: Se dispara cuando el cursor entra en un elemento
  • mousemove: Se dispara mientras el cursor pasa sobre un elemento
  • mouseout: Se dispara cuando el cursor sale de un elemento
  • otros: drag, drop, dragstart, dragend...

Eventos HTML

  • load: Se dispara una vez que se ha cargado todo el elemento, incluyendo frames, imagenes, etc.
  • unload: Se dispara una vez que el user agent ha eliminado todos los elementos de la ventana o frame
  • error: Se dispara una vez que una imagen/objeto/frame no se carga correctamente
  • abort: Se dispara una vez que se detiene la carga de una imagen/objeto/frame antes de cargarse completamente

Eventos específicos de Formulario

  • select: Aparece cuando se selecciona un texto en un campo de texto.
  • focus: Aparece cuando un elemento del formulario recibe el foco, bien sea mediante ratón o bien por tabulador.
  • blur: Aparece cuando un elemento del formulario pierde el foco, bien sea mediante ratón o bien por tabulador.
  • change: Aparece cuando un input pierde el foco y su valor ha cambiado
  • submit: Aparece cuando se hace submit en el formulario
  • reset: Aparece cuando se hace reset en el formulario

Diferencia entre .on(), .live(). bind() y .delegate()

bind() Deprecated

Es utilizado para definir cualquier tipo de evento, pudiendo definir varios tipos de eventos de una sola vez. Sin embargo, solo funciona para los elementos coincidentes con el selector existentes en el momento de la ejecución del método bind.

$("p").bind("click", function(event) { /*codigo*/ });

live() Deprecated

En contrasate con bind(), live(), si que permite definir los eventos tanto para elementos que actualmente coinciden con el selector como para aquellos que no existen pero lo harán en el futuro, por lo que recorrerá el arbol DOM varias veces.

$("p").live("click", function(event) { /*codigo*/ });

delegate() Deprecated

Asigna un manejador a uno o mas eventos para los elementos que cumplan el selector, tanto si están en el presente como si lo harán en el futuro

$("main").delegate("p","click", function(event) { /*codigo*/ });

on()

Define una función hacia uno o más eventos para los elementos seleccionados. Internamente es la función que utiliza jquery con otros métodos como click. Aparece como una conversión de los métodos bind, delegate y live.

$("main").on("a","click", function(event) { /*codigo*/ });
